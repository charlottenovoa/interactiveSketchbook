Micro Project Self Assessment

Core Rule & System Clarity

20 pts: The rule for the sketch is when the mouse is pressed, a bite is taken. This describes the whole premise of the ice cream being eaten as the user interacts.

State Progression

~13pts: Ice cream can be eaten multiple times, but as overarching states, I think there's only 2; the ice cream as it's eaten and the 'I'm full' message. If each bite being taken is a different state then my answer changes (lol!). (I added an extra state since writing this section. I forgot to add it to the file version before submitting so I put it up one more point!)

Interaction and Evolution

15pts: The whole sketch is based on user interaction. If the user does nothing, the sketch does not progress. When they do interact, a random number of bites (within a range) is taken from the ice cream. Over time this accumulates accross the screen.

Iterative Development

14pts?: From the first iteration, I believe the sketch has changed a lot. I struggled a lot with the earlier versions, but finally came to a point where it worked more to my original intentions. I orginally used an array of photos to atempt to show the bites, but that meant the placement was predetermined and only one flavor could be eaten. Now, the bite placement is dependent on where the user clicks, and there's actually a message for when the user is full which is randomized.

Documentation & Logging

15pts: All of my sketches are catalogged on this site. Including all of my progress reports on my micro sketch.

Risk & Experimentation
