Interface Audit

Visibility of Systems Status

The page I chose to audit was the landing page as it contains all of the navigation for the sketches and other pages (reading responses, progress checks, etc). The page doesn’t say anything like “homepage” or anything on it, which maybe could be added. However, most sites don’t tell you that you’re on the homepage rather they just make it clear where you are. Perhaps more could be added to illustrate that. The progress checks bar does indicate that it can be interacted with because it does just come out and tell you. The arrows for the sketches do seem like buttons, but maybe aren’t obvious as to what they navigate to. The weak point for this page is needing more clear navigation, and more clearly showing that the user is on the homepage.

Affordances & Signifiers

The index is really only there for being a navigation hub. Therefore, it only has the top bar and the arrows. These additions are based on things that any website could have like a drop down menu, or the arrow buttons. Because it’s only those items, there isn’t anything that goes unnoticed. Like I said previously, more explanation as to what is happening with those buttons would be something I would add.

Navigation & Mapping

I had trouble fitting the amount of sketches, and the extra pages, so I spilt them into two separate navigation systems. The unfortunate result of this is that the sketches are only reachable from the homepage. This may cause some issues where the user is on one of the reading response pages and can’t get to my sketches. Especially because it causes them to need to remember that the only way to navigate to them is on the main page. The arrows also don’t say what they’re for, so it can cause some confusion. The drop down menu however, is more clear as to what it’s for as everything is labeled and can be seen what is going to appear before it is clicked on.

Constraints & Assumptions

One thing that the user would have to assume is what the arrows do. I know as the person who made it that the arrows are going through the sketches 1-9, but the user may not know that. Because of that, they may need to guess before using them. This is where the hesitation could come into play.

Heuristic Violations

Of the heuristics, I think my page violates six and eight. I mentioned before about the arrows only being on the homepage. It does require the user to remember where the buttons are and what they do. The site violates eight because of the way I’ve laid out the navigation. Aesthetically it’s a bit strange. Not so much in terms of colors/fonts, but how the lay out impacts usability.

Structural Revision Proposal

  1. Make the navigation more condensed and accessible.
  2. Clarify what each thing does.
  3. Add more to the homepage (make it look more like homepage)
