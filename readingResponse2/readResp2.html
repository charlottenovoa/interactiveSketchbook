Reading Response #2

Of the ten heuristics, one and two seem most aligned with Norman’s concerns. He says that “Two of the most important characteristics of good design are discoverability and understanding. Discoverability: Is it possible to even figure out what actions are possible and where and how to perform them? Understanding: What does it all mean? How is the product supposed to be used? What do all the different controls and settings mean?”. To me this relates to the first heuristic on being informed on the system. Nielsen suggests that we should keep the user informed on the current state of the system, similar to how Norman wants the user to be able to discover and see what is possible in the system. The second heuristic relates to the language not being over complicated and understandable to the consumer. As Norman says, it is important to consider how this system is meant to be used and what we can do with it, without much extra effort.

In my opinion, my micro project lacks on the first and the tenth heuristic. The first one says that the user should always be aware of the current state of the system. My micro projects states can be a little unclear. There is a message indicating if the ice cream is finished or unfinished, but the finished message does not appear when the ice cream is truly done so it’s slightly confusing. The tenth heuristic is about the user not needing any extra explanation. I think that my micro project could benefit from some explanation of the circumstances that they are placed in. The sketch just starts and ends. If I were to add this, I would likely add a start screen or more levels with more descriptive controls and information. These are important because it affects how the user is able to use my sketch. If the states are unclear, people will not want to use it. Similarly, the lack of description does not make it any more usable or interesting.

For my sketchbook, I think it lacks on the aesthetic and recognition rather than recall heuristics. The visuals are simple, but in a way that it needs more to be more appealing and accessible to a user. It affects how legible everything is and how interact-able it is. To the point on recognition. How I made my sketches vs. my other items (my progress checks, reading responses, etc) accessible is a bit confusing. I know that you have to navigate to the homepage to find the buttons to flip through the sketches, but it may be inconvenient or hard to recall for those who are navigating for the first time. Actions are reversible. No page has a way to get stuck, as every page can lead to the homepage. In addition, everything has a title, so the user should know where they are without much thought. Overall, I think I would implement a different kind of navigation system, as well as more easy to look at page design.
